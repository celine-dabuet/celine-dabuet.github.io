Problem Solving

Blocked on a simple problem

The simple problem I had was figuring why the solution I provided wasn't accepted. Regardless of providing the correct answer, the test failed me and this was a real mystery because it was a good reminder that no matter how confident I was about my code, there was something that could be wrong. Checking the logic of my code line by line was always the first thing I did; making sure there were no typos, accidental capitalization or lack thereof of my wording, dissecting each code and making sure I understood every single character I used. It was really frustrating solving this simple problem because I found this specific challenge to be a breeze to do and I quickly understood how the built-in methods in Javascript are supposed to be used, so to be stumped on why the test was failing me was mind-boggling. Upon checking the questions and staring at the checking requirements for a long period of time that it was then that I realised I wasn't fulfilling the requirements of the question on how the solution is provided. In coding, there are different ways to print a result onto the console/browser. First, there is the straightforward, console.log() method, and second there is the return keyword. In this case, I constantly did the former rather than the latter option. I learned that carefully understanding all the components of the question is really important as it's not always about giving the correct answer but how it's given.

Elegantly solved a problem

A very peculiar question as I have no idea how I can elegantly solve a problem. I don't know if there's anything elegant in glaring at a screen and talking loudly at a notebook. But in terms of making the code more sophisticated, I have learned the shorthand way of writing functions using the arrow => method. I'm still wrapping my head around it fully but I realised that I should aim to make my code as short as possible and as less-hard-coded as I can make it. When I was able to use this shorthand version in the Javascript Kata challenge, I think the code looked quite dashing seeing it in its most succinct form.
